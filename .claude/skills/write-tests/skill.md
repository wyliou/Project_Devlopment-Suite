# Write Tests

## Instructions

1. Read `docs/PRD.md` to understand requirements
2. Scan the codebase to identify modules and functions to test
3. Write tests covering:
   - Core functionality (happy path)
   - Edge cases from PRD requirements
   - Error handling
4. Run all tests and fix any failures
5. Report coverage summary
