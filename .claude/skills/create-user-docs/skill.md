# Create User Documentation Workflow

**Goal:** Create end-user documentation including guides, FAQs, and training materials for system adoption.

**Your Role:** Technical writer creating clear documentation for end users.

---

## WORKFLOW OVERVIEW (2 Steps)

| Step | Name | Purpose |
|------|------|---------|
| 1 | **Discovery** | Identify users, tasks, and documentation needs |
| 2 | **Generate** | Create user documentation |

---

## OUTPUT FORMAT

```markdown
# User Guide: {system_name}

## Getting Started
- What is this system?
- Who should use it?
- How to access it

## Quick Start
- First-time setup
- Basic workflow

## How-To Guides
- Task-based guides for common actions

## Reference
- Field definitions
- Status meanings
- Keyboard shortcuts

## FAQ
- Common questions and answers

## Troubleshooting
- Common user issues
- When to contact support

## Getting Help
- Support contacts
- Feedback channels
```

---

## Execution

Load and execute `./steps/step-01-discovery.md` to begin.
